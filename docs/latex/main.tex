\section{Analiza problemu}
% =========================================================

\subsection{Opis problemu}

Studenci muszą pamiętać o wielu zadaniach, projektach i terminach. Informacje te są często zapisywane w różnych miejscach, przez co łatwo o nich zapomnieć.

StudyPlanner pozwala zapisać wszystkie zadania w jednej aplikacji. Użytkownik może sprawdzić termin, priorytet oraz aktualny status zadania.

Aplikacja porządkuje zadania i ułatwia kontrolowanie postępów w nauce.

\subsection{Użytkownik aplikacji}

Aplikacja jest przeznaczona głównie dla studentów i osób uczących się.

Użytkownik potrzebuje możliwości:

\begin{itemize}[leftmargin=*]
    \item zapisania zadania,
    \item określenia terminu,
    \item wybrania priorytetu,
    \item zmiany statusu,
    \item edycji lub usunięcia zadania,
    \item wyszukania zadania,
    \item wyświetlenia tylko własnych danych.
\end{itemize}

\section{Analiza funkcjonalna}
% =========================================================

\subsection{Lista funkcji}

Aplikacja posiada następujące funkcje:

\begin{itemize}[leftmargin=*]
    \item rejestracja użytkownika,
    \item logowanie i wylogowanie,
    \item zapamiętywanie sesji,
    \item dodawanie zadań,
    \item edytowanie zadań,
    \item usuwanie zadań,
    \item oznaczanie zadania jako wykonane,
    \item ustawianie statusu,
    \item ustawianie priorytetu,
    \item wyszukiwanie zadań,
    \item filtrowanie po statusie,
    \item filtrowanie po priorytecie,
    \item wyświetlanie statystyk,
    \item wykrywanie zadań po terminie,
    \item walidacja formularzy,
    \item obsługa błędów i pustej listy.
\end{itemize}

\subsection{User stories}

\begin{itemize}[leftmargin=*]
    \item Jako użytkownik chcę utworzyć konto, aby korzystać z własnego planera.
    \item Jako użytkownik chcę się zalogować, aby zobaczyć swoje zadania.
    \item Jako student chcę dodać zadanie, aby zapisać obowiązek.
    \item Jako student chcę ustawić termin, aby wiedzieć, kiedy wykonać zadanie.
    \item Jako student chcę ustawić priorytet, aby określić ważność zadania.
    \item Jako student chcę zmienić status, aby kontrolować postęp.
    \item Jako student chcę edytować zadanie, aby poprawić jego dane.
    \item Jako student chcę usunąć zadanie, które nie jest już potrzebne.
    \item Jako student chcę filtrować zadania, aby szybciej znaleźć potrzebne wpisy.
    \item Jako użytkownik chcę widzieć tylko własne zadania.
\end{itemize}

\subsection{Model danych}

Zadania są przechowywane w tabeli \texttt{tasks}.

\begin{table}[H]
\centering
\begin{tabularx}{\textwidth}{|p{3cm}|p{2.8cm}|X|}
\hline
\textbf{Pole} &
\textbf{Typ} &
\textbf{Opis} \\
\hline
id & UUID & Identyfikator zadania. \\
\hline
user\_id & UUID & Identyfikator właściciela zadania. \\
\hline
title & TEXT & Nazwa zadania. \\
\hline
description & TEXT & Opcjonalny opis zadania. \\
\hline
subject & TEXT & Nazwa przedmiotu. \\
\hline
deadline & DATE & Termin wykonania. \\
\hline
priority & TEXT & Priorytet: niski, średni lub wysoki. \\
\hline
status & TEXT & Status: do zrobienia, w trakcie lub wykonane. \\
\hline
created\_at & TIMESTAMP & Data utworzenia zadania. \\
\hline
updated\_at & TIMESTAMP & Data ostatniej zmiany. \\
\hline
\end{tabularx}
\caption{Struktura tabeli tasks}
\end{table}

Każde zadanie posiada pole \texttt{user\_id}. Dzięki temu jest przypisane do konkretnego użytkownika.

Tabela korzysta z Row Level Security. Użytkownik może odczytywać i zmieniać tylko własne zadania.

\subsection{Architektura aplikacji}

Struktura projektu:

\begin{verbatim}
study-planner/
|-- public/
|-- src/
|   |-- main.js
|   |-- style.css
|   |-- tasks.css
|   `-- supabase.js
|-- docs/
|   `-- screenshots/
|-- index.html
|-- package.json
|-- package-lock.json
|-- .gitignore
`-- README.md
\end{verbatim}

Najważniejsze pliki:

\begin{itemize}[leftmargin=*]
    \item \texttt{index.html} -- podstawowa struktura aplikacji,
    \item \texttt{main.js} -- logowanie, rejestracja i obsługa zadań,
    \item \texttt{style.css} -- wygląd logowania i rejestracji,
    \item \texttt{tasks.css} -- wygląd dashboardu i formularza zadania,
    \item \texttt{supabase.js} -- konfiguracja połączenia z Supabase.
\end{itemize}

\subsection{Przepływ działania aplikacji}

\begin{enumerate}[leftmargin=*]
    \item Użytkownik otwiera aplikację.
    \item Aplikacja sprawdza, czy istnieje aktywna sesja.
    \item Niezalogowany użytkownik widzi formularz logowania.
    \item Użytkownik loguje się lub zakłada konto.
    \item Po zalogowaniu pobierane są zadania użytkownika.
    \item Użytkownik otwiera formularz zadania.
    \item Formularz sprawdza poprawność danych.
    \item Dane są zapisywane w Supabase.
    \item Lista zadań i statystyki są aktualizowane.
    \item Użytkownik może edytować, usuwać lub filtrować zadania.
\end{enumerate}

\subsection{Uruchomienie lokalne}

\begin{verbatim}
git clone https://github.com/tolpamaksymilian/study-planner.git
cd study-planner
npm install
npm run dev
\end{verbatim}

W głównym folderze projektu należy utworzyć plik \texttt{.env.local}:

\begin{verbatim}
VITE_SUPABASE_URL=ADRES_PROJEKTU_SUPABASE
VITE_SUPABASE_PUBLISHABLE_KEY=KLUCZ_PUBLISHABLE
\end{verbatim}

Po uruchomieniu aplikacja jest dostępna pod adresem:

\url{http://localhost:5173}
